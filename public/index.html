Your friends, in your statusline.

One extra line under your Claude Code statusline, refreshed every 30 seconds:

● 🛸 marco · fighting a flaky test (side-project)

● 🌸 lisa · tuning the telegram alerts (apartment-bot)

○ 🐙 joao · idle (zine-gen) · 2h

marco, lisa and joao are the demo accounts. Your notch shows your actual friends — added by invite code only, never discoverable by username.

The server never sees your prompts.

When you prompt Claude, a hook posts three things: a working/idle flag, your project's folder name, and a one‑line summary. The summary is built on your machine by summarize.js — first line only, code blocks dropped, credentials, links, emails and paths redacted, cut at 80 characters. That sanitized line is the only prompt-derived text that ever goes over the wire, and the sanitizer is ~60 dependency-free lines so you can audit the whole boundary in one sitting.

Want to share less? presence share project sends the folder name only. presence ghost on sends nothing at all.

Install

  1. 01

    Tell your Claude.

    Paste this into Claude Code. It reads the repo, asks for your username and emoji, and wires the notch into every new session.

    paste into claude code
    Set up claude-presence for me — read
https://github.com/limone-eth/claude-presence
and follow the README install instructions.
  2. 02

    Or run it by hand.

    The installer asks for your username, an emoji, and an optional invite code, right in the terminal. It backs up your Claude settings before touching them.

    shell
    $ curl -fsSL https://raw.githubusercontent.com/limone-eth/claude-presence/main/bin/bootstrap.sh | bash
  3. 03

    Befriend someone.

    Mint a code with presence invite and send it over whatever channel you already share. One code, one friendship — single-use, expires in 7 days. Their notch lights up with you; yours with them.
