# terminally.social

## Register
brand — the landing page IS the product surface; design carries the pitch.

## Product
Discord-style presence + a daily token-usage leaderboard ("tokenmaxxing") rendered inside the
Claude Code statusline. Friends see what you're building and who burned the most tokens today;
the crown resets at midnight UTC. Privacy invariant: prompts are sanitized on-device and never
reach the server (client/summarize.js is the auditable boundary).

## Users
Claude Code power users and terminally-online developers, arriving from links their friends
dropped in group chats. They pattern-match AI-generated design instantly and respect terminal
craft. One action matters: copy the install prompt into Claude Code.

## Brand personality
competitive, ironic, technical.

## Anti-references
- AI-slop scaffolding: rounded accent-border cards, icon feature grids, hero-metric templates,
  eyebrow kickers on every section, cream-paper editorial defaults.
- Matrix/hacker clichés: phosphor-green glyph rain, CRT scanlines (a shader was tried and removed).
- SaaS landing rhythm: hero → 3 features → CTA → fat link footer.

## Design principles
- The page is one terminal session: a single no-scroll fold; the demo is real product output
  (statusline, notch, `└ Tip:` line), never re-drawn chrome.
- Poster typography on the left, quiet terminal artifact on the right; the tension between
  loud display and calm mono is the composition.
- One warm amber accent (the page must not read green-on-black); status green survives only as
  the working dot. JetBrains Mono display + Geist body, nothing else.
- Copy is deadpan competitive shitposting with zero dashes; the privacy line stays subtle
  (footer) because it's serious.
- Honest content only: real demo accounts, no invented metrics or testimonials.
